## Part 1: VS Code â€” Your Code Editor

VS Code is where you'll spend most of your time. It's free, fast, and has an incredible ecosystem of extensions.

<Steps>
<Step title="Download and Install">

1. Go to [code.visualstudio.com](https://code.visualstudio.com)
2. Click **Download for Mac**
3. Open the downloaded `.zip` file
4. Drag **Visual Studio Code.app** to your Applications folder

<Callout type="note">
If you see a security warning, go to System Preferences â†’ Security & Privacy â†’ Open Anyway.
</Callout>

</Step>
<Step title="Enable Terminal Command">

This lets you open VS Code by typing `code` in the terminalâ€”a huge time saver.

1. Open VS Code
2. Press <Kbd>Cmd</Kbd> + <Kbd>Shift</Kbd> + <Kbd>P</Kbd> (Command Palette)
3. Type **shell command**
4. Select **Shell Command: Install 'code' command in PATH**

**Test it** â€” Open Terminal and run:

```bash
code --version
```

You should see a version number like `1.85.0`.

</Step>
</Steps>

## Part 2: Homebrew â€” Your Mac's Missing App Store

Ever downloaded a sketchy `.pkg` file from a random website to install a developer tool? Homebrew fixes that. It's how professional Mac developers install everything.

<Diagram type="flowchart" caption="Why Homebrew is better">
{`flowchart LR
    subgraph old["Without Homebrew"]
        A1[Google the tool] --> A2[Find download page]
        A2 --> A3[Download .pkg/.dmg]
        A3 --> A4[Run installer]
        A4 --> A5[Manual updates forever]
    end

    subgraph new["With Homebrew"]
        B1[brew install tool] --> B2[Done]
        B2 --> B3[Auto-updates]
    end
`}
</Diagram>

<Steps>
<Step title="Install Homebrew">

Open **Terminal** (press <Kbd>Cmd</Kbd> + <Kbd>Space</Kbd>, type "Terminal", press Enter) and paste this command:

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

<Callout type="terminal">
You'll be asked for your password. Type it (nothing appearsâ€”that's a security feature) and press Enter. Then follow any prompts.
</Callout>

</Step>
<Step title="Add to PATH">

After installation, Homebrew shows you some commands to run. They look like this:

```bash
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"
```

Run both commands, then verify:

```bash
brew --version
```

You should see something like `Homebrew 4.2.0`.

</Step>
</Steps>

## Part 3: Git â€” Never Lose Your Work Again

Git tracks every change you make to your code. Made a mistake? Go back. Want to experiment? Create a branch. Collaborating? Merge your changes.

<Callout type="tip">
Think of Git as an "infinite undo" for your entire projectâ€”but better, because you can also see *what* you changed and *why*.
</Callout>

<Steps>
<Step title="Install Git">

```bash
brew install git
```

That's it. Homebrew handles the rest.

</Step>
<Step title="Configure Your Identity">

Git tags every change with your name and email. Set these once:

```bash
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
```

<Callout type="warning">
Use the same email you'll use for GitHub. This links your commits to your GitHub profile.
</Callout>

</Step>
<Step title="Set Sensible Defaults">

```bash
# Use VS Code for commit messages
git config --global core.editor "code --wait"

# Default branch name (GitHub standard)
git config --global init.defaultBranch main

# Handy shortcuts
git config --global alias.st status
git config --global alias.co checkout
git config --global alias.br branch
```

Verify everything:

```bash
git config --list
```

</Step>
</Steps>

## Part 4: GitHub â€” Your Code's Home in the Cloud

GitHub is where your code lives online. It's also where you'll collaborate with others and showcase your work.

<Diagram title="Local vs Remote" type="flowchart" caption="Git works locally; GitHub stores your code in the cloud">
{`flowchart TB
    subgraph local["Your Mac"]
        A[Your Code] --> B[Git tracks changes]
    end

    subgraph cloud["GitHub"]
        C[Remote Repository]
    end

    B -->|git push| C
    C -->|git pull| B
`}
</Diagram>

<Steps>
<Step title="Create a GitHub Account">

1. Go to [github.com](https://github.com)
2. Click **Sign up**
3. Pick a professional username (this shows up on your profile and contributions)

</Step>
<Step title="Generate SSH Key">

SSH keys let you push code to GitHub without entering your password every time.

```bash
ssh-keygen -t ed25519 -C "your.email@example.com"
```

When prompted:
- **File location**: Press Enter (accept default)
- **Passphrase**: Type one (or press Enter for none)
- **Confirm**: Press Enter

</Step>
<Step title="Start SSH Agent">

```bash
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
```

</Step>
<Step title="Add Key to GitHub">

Copy your public key:

```bash
pbcopy < ~/.ssh/id_ed25519.pub
```

Now add it to GitHub:

1. Go to [github.com/settings/keys](https://github.com/settings/keys)
2. Click **New SSH key**
3. Title: "My Mac" (or whatever you want)
4. Key type: **Authentication Key**
5. Paste with <Kbd>Cmd</Kbd> + <Kbd>V</Kbd>
6. Click **Add SSH key**

</Step>
<Step title="Test the Connection">

```bash
ssh -T git@github.com
```

<Callout type="success">
You should see: **"Hi [username]! You've successfully authenticated..."**
</Callout>

If you see a warning about authenticity, type `yes` and press Enter.

</Step>
</Steps>

## Part 7: Python (For Data Analysis)

<Callout type="note">
**Skip this** if you're not planning to do data analysis. You can always come back later.
</Callout>

<Steps>
<Step title="Install Python">

```bash
brew install python@3.11
```

</Step>
<Step title="Create a Virtual Environment">

Virtual environments keep each project's packages separate (trust us, you want this):

```bash
mkdir ~/my-first-project
cd ~/my-first-project
python3 -m venv venv
source venv/bin/activate
```

Your prompt now shows `(venv)` â€” you're in the virtual environment.

</Step>
<Step title="Install Data Science Packages">

```bash
pip install pandas numpy matplotlib jupyter
```

When done working:

```bash
deactivate
```

</Step>
</Steps>

## Troubleshooting

<Tabs>
  <TabList>
    <Tab>Command Not Found</Tab>
    <Tab>Permission Denied</Tab>
    <Tab>SSH Issues</Tab>
    <Tab>VS Code + Git</Tab>
  </TabList>
  <TabPanel>

### "command not found"

This usually means the tool isn't in your PATH. Try:

1. **Close and reopen Terminal** â€” PATH updates require a new session
2. **Check Homebrew PATH** â€” Run the commands from Part 2, Step 2 again
3. **Verify installation** â€” `brew list | grep [tool-name]`

  </TabPanel>
  <TabPanel>

### Permission Denied (npm)

If npm gives permission errors when installing global packages:

```bash
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.zprofile
source ~/.zprofile
```

  </TabPanel>
  <TabPanel>

### SSH Won't Connect

If `git push` asks for username/password:

1. **Check remote URL** â€” Should start with `git@github.com:`, not `https://`
   ```bash
   git remote -v
   ```

2. **Fix if needed**:
   ```bash
   git remote set-url origin git@github.com:username/repo.git
   ```

3. **Verify SSH key** â€” Repeat Part 4 (Generate and add SSH key)

  </TabPanel>
  <TabPanel>

### VS Code Can't Find Git

If VS Code shows "Git not found":

1. Open Settings (<Kbd>Cmd</Kbd> + <Kbd>,</Kbd>)
2. Search for `git.path`
3. Set it to `/opt/homebrew/bin/git`

  </TabPanel>
</Tabs>
