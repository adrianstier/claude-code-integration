## What Makes This Different

This isn't "learn pandas methods" or "memorize ggplot syntax." You'll work on **real data problems** with Claude as your coding partner.

<Callout type="tip">
**Claude writes the code. You understand the data.** That's the workflow. You describe what you need, Claude generates it, and you learn by doing.
</Callout>

## Python vs R: Pick Your Tool

<Tabs>
  <TabList>
    <Tab>Python</Tab>
    <Tab>R</Tab>
  </TabList>
  <TabPanel>
**Python** is the Swiss Army knife of data analysis.

Best for: Machine learning, web scraping, automation, production systems, general-purpose programming.

You'll use: pandas, matplotlib, seaborn, scikit-learn, Jupyter notebooks.

**Who chooses Python:**
- Data scientists in tech companies
- Anyone doing machine learning
- People who want one language for everything
- Developers who also need to build apps

[Start Python Track ‚Üí](/data-analysis/python-intro)
  </TabPanel>
  <TabPanel>
**R** was built by statisticians, for statisticians.

Best for: Statistical analysis, data visualization, academic research, reporting.

You'll use: tidyverse, ggplot2, dplyr, RMarkdown.

**Who chooses R:**
- Researchers and academics
- Anyone doing serious statistical analysis
- Data journalists
- People who love beautiful visualizations

[Start R Track ‚Üí](/data-analysis/r-intro)
  </TabPanel>
</Tabs>

<Callout type="note">
**Can't decide?** Start with Python. It's more versatile, and the skills transfer to R easily. You can always learn R later for specialized statistical work.
</Callout>

## The Workflow (Both Tracks)

Every data project follows the same pattern:

<Diagram title="Analysis Workflow" type="flowchart" caption="This cycle repeats constantly">
{`flowchart TB
    A[1. Ask a question] --> B[2. Get the data]
    B --> C[3. Clean it]
    C --> D[4. Explore]
    D --> E{Found insight?}
    E -->|No| A
    E -->|Yes| F[5. Visualize]
    F --> G[6. Document]
    G --> H[7. Share]
`}
</Diagram>

### How Claude Fits In

At each step, Claude helps you:

| Step | What Claude Does |
|------|------------------|
| **Get data** | Write import code, handle file formats |
| **Clean** | Generate cleaning code, explain why |
| **Explore** | Suggest analyses, write aggregations |
| **Visualize** | Create charts, customize styling |
| **Model** | Build models, explain results |
| **Document** | Write reports, explain findings |
